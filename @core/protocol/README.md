# @core/type

规定所有数据协议的基础结构。

## 结构

所有数字均为网络字节序（大端序）。

| 字段 | 类型/长度(字节) | 描述 |
| --- | --- | --- |
| frame_length | uint32 | 帧长度，表示整个数据包的字节数，包括结尾END，不包括帧长度自身 |
| START | uint8 * 5 | 起始标志，固定为 'START' ，验证逻辑是否正确 |
| version | uint8 | 协议版本号，当前版本为 `1` |
| sender_agent_length | uint8 | sender_agent 字段的长度，最长255 |
| sender_agent | ... | 发送者类型，表示数据包的来源，调试用 |
| sender_id | uint32 | 发送者ID，表示数据包的来源，调试用 |
| action | uint32 | 行为码（见下） |
| payload_length | uint32 | 数据长度，表示后续数据的字节数，不包括前面所有字段 |
| payload | ... | 数据包 |
| END | uint8 * 3 | 结束标志，固定为 'END' ，验证逻辑是否正确 |

## 行为

| 行为码 | 描述 |
| --- | --- |
| 0x1 | 保持连接，每个程序都会随机收到此包，必须回应 |
| 0x2 | 保持连接的回应 |
| 0x3 | 数据帧，包含传感器数据等信息 |

以下为各种行为的payload结构：

#### 保持连接/回应

| 字段 | 类型/长度(字节) | 描述 |
| --- | --- | --- |
| timestamp | uint64 | 时间戳（**毫秒**），没有实际作用 |

需要尽快回应。    
可以设置timestamp字段为当前时间，但不是必须的。

#### 数据帧

| 字段 | 类型/长度(字节) | 描述 |
| --- | --- | --- |
| function | uint32 | 功能码，表示具体的操作或命令，各程序自己定义，也可不用（设为0） |
| timestamp | uint64 | 时间戳，表示数据包所包含数据的**开始**时间，单位为**毫秒** |
| type | uint8 | 符号和类型，`'u'(0x75)` 表示无符号整数，`'s'(0x73)` 表示有符号整数，`'f'(0x66)` 表示有符号浮点数 |
| bit_depth | int8 | 位深（见下方） |
| rate | uint32 | 采样率，每**秒**点数 |
| header | uint32 | 固定为 0x41444154 (`DATA`)，验证逻辑是否正确 |
| array | ... | 数据内容 |

### 关于位深的说明

可取值：8、16、32

决定payload应该如何解释，例如：
> 如果bit_depth为16，payload中的每个数据点占用2字节。即 uint16 数组。 **再次注意，大端序**

对于任意一套系统，理论上不应该存在多个bit_depth的情况，该值基本取决于传感器采样精度。
