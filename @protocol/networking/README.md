# @protocol/base

规定所有数据协议的基础结构。

## 结构

所有数字均为网络字节序（大端序）。

| 字段 | 类型/长度(字节) | 描述 |
| --- | --- | --- |
| frame_length | uint64 | 帧长度，表示整个数据包的字节数，包括结尾 |
| version | uint8 | 协议版本号，当前版本为 `1` |
| sender_agent | uint64 | 发送者类型，表示数据包的来源，调试用 |
| sender_id | uint32 | 发送者ID，表示数据包的来源，调试用 |
| action | uint64 | 行为码（见下） |
| payload_length | uint32 | 数据长度，表示后续数据的字节数，不包括前面所有字段 |
| payload | ... | 数据包 |
| END | uint8 * 2 | 结束标志，固定为0xBE 0xBF，验证逻辑是否正确 |

### 关于位深的说明

可取值：8、16、32

决定payload应该如何解释，例如：
> 如果bit_depth为16，payload中的每个数据点占用2字节。即 uint16 数组。 **再次注意，大端序**

对于任意一套系统，理论上不应该存在多个bit_depth的情况，该值基本取决于传感器采样精度。

## 行为

| 行为码 | 描述 |
| --- | --- |
| 0x1 | 保持连接，每个程序都会随机收到此包，必须回应 |
| 0x2 | 保留 |
| 0x3 | 数据帧，包含传感器数据等信息 |

以下为各种行为的payload结构：

#### 保持连接

| 字段 | 类型/长度(字节) | 描述 |
| --- | --- | --- |
| timestamp | uint64 | 时间戳（**毫秒**），没有实际作用 |

需要尽快回应，除了sender_agent和sender_id之外，其他字段都和收到的包一样即可。


#### 数据帧

| 字段 | 类型/长度(字节) | 描述 |
| --- | --- | --- |
| function | uint64 | 功能码，表示具体的操作或命令，各程序自己定义，也可不用（设为0） |
| timestamp | uint64 | 时间戳，表示数据包所包含数据的**开始**时间，单位为**毫秒** |
| signed | uint8 | 是否有符号，`'u'(0x75)` 表示无符号，`'s'(0x73)` 表示有符号 |
| bit_depth | int8 | 位深（见下方） |
| is_interger | uint8 | 是否为整数，`'f'(0x66)` 表示浮点数，`'i'(0x69)` 表示整数 |
| rate | uint32 | 采样率，每**秒**点数 |
| array | ... | 数据内容 |
